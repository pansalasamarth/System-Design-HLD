# High Level Design (HLD)

This repository contains my **structured notes for High Level Design (HLD)** preparation.  
The goal of this repository is to build a **strong foundation in designing scalable, reliable, and distributed systems**.

> **"This is one of the things that helped me understand real-world engineering systems."**

High Level Design is one of the most important skills for software engineers because it focuses on **how large-scale systems are architected, scaled, and maintained in production**.

While **Low Level Design (LLD)** focuses on classes, objects, and code structure, **HLD focuses on services, databases, APIs, scalability, and system architecture**.

---

## Why HLD is Important

- Helps design **scalable distributed systems**
- Improves **architecture thinking skills**
- Teaches **tradeoffs in real-world systems**
- Helps understand **databases, caching, queues, and networking**
- Essential for **backend and senior engineering roles**
- Frequently asked in **software engineering interviews**

> **"If DSA is the brain, HLD is the nervous system of your application."**

---

## Prerequisites

Before starting HLD, having a basic understanding of these subjects is highly recommended:

- **Operating Systems (OS)** – Processes, threads, concurrency, memory management  
- **DBMS** – SQL, indexing, normalization, transactions  
- **Computer Networks (CN)** – HTTP, TCP/IP, DNS, load balancing basics  
- **Object-Oriented Programming (OOPs)** – Classes, objects, inheritance, polymorphism, SOLID principles  

> **"Strong fundamentals make system design easier to understand."**
